# 02 — 上传

## 生成 upload session

在 `projects/qiniu-uploader/` 目录执行（AK/SK 建议用环境变量，勿提交仓库）：

```bash
cd projects/qiniu-uploader/scripts

python3 qiniu_prefix_uptoken.py \
  --access-key "$QINIU_ACCESS_KEY" \
  --secret-key "$QINIU_SECRET_KEY" \
  --bucket "bizdl-airdroid-com" \
  --key-prefix "videos/task123/device456/" \
  --upload-url "https://up-z2.qiniup.com" \
  --expires-in 1800 \
  --fsize-limit 5368709120

//真实
python3 qiniu_prefix_uptoken.py \
  --access-key "fpWzUFhgkUGVEph0rXeBVu6Y4hfMhuZ5qXqHYjJi" \
  --secret-key "1AsXkvDBlfVrl1NU89Q7InrYZ9GMjGu8Jlnefuqk" \
  --bucket "bizdl-airdroid-com" \
  --key-prefix "videos/task123/device456/" \
  --upload-url "https://up-z2.qiniup.com" \
  --expires-in 1800 \
  --fsize-limit 5368709120   
```

输出 JSON 字段含义：

| 字段 | 含义 |
|------|------|
| `uploadUrl` | 表单 POST 目标，必须与 bucket 机房一致 |
| `uploadToken` | 表单字段 `token` 的值 |
| `keyPrefix` | 客户端拼 `key` 时必须落在此前缀下 |
| `expiresAt` / `expiresInSec` | 过期时间 |
| `constraints` | 可选，给前端展示的 MIME/大小限制 |

### 可选参数

- `--mime-limit 'video/*'`：限制 MIME；前期试传 PDF 等**不要加**。
- `--callback-url` / `--callback-body`：上传成功回调；**前期可全部省略**。
- `--insert-only`：禁止覆盖同名 key。

## 最本质的上传 curl

七牛表单上传 = **POST + multipart/form-data**，三个字段：

| 字段 | 说明 |
|------|------|
| `token` | 完整 uploadToken |
| `key` | 完整对象路径，须落在 `keyPrefix` 下 |
| `file` | 本地文件 |

```bash
curl -v -X POST 'https://up-z2.qiniup.com' \
  -F 'token=PASTE_UPLOAD_TOKEN_HERE' \
  -F 'key=videos/task123/device456/滴滴电子发票.pdf' \
  -F 'file=@"/path/to/滴滴电子发票.pdf"'
```

注意：

- **不要**手写 `Content-Type: multipart/form-data; boundary=...`（Paw/Postman 导出常带错 boundary）。
- **不要** `-F 'file='` 空文件。
- `key` 含中文时，curl 的 `-F` 一般可处理；脚本侧会对文件名做安全化（见 `upload_local_directory_to_qiniu.py`）。

成功响应示例：

```json
{
  "hash": "Ft_Bx_QC1xyoCGsiApq-fVw9MRhq",
  "key": "videos/task123/device456/滴滴电子发票.pdf"
}
```

## 上传域名与机房

七牛会按 bucket 所在区域校验上传入口。错误示例：

```json
{"error":"incorrect region, please use up-z2.qiniup.com, bucket is: bizdl-airdroid-com"}
```

表示应使用华南上传域 `https://up-z2.qiniup.com`，而不是华东 `https://upload.qiniup.com`。

常见上传域名（以控制台 / 文档为准）：

| 区域 | 常见上传域名 |
|------|----------------|
| 华东 z0 | `https://upload.qiniup.com` |
| 华南 z2 | `https://up-z2.qiniup.com` |

`session` 里的 `uploadUrl` 必须与签发时一致，客户端原样 POST 即可。

## uploadToken 签名注意点

`qiniu_prefix_uptoken.py` 生成 token 时：

- 对 policy 做 URL 安全 Base64 时**保留末尾 `=` padding**（与官方 SDK 一致）；去掉 padding 会导致 `401 BadToken`。
- 签名原文是 **Base64 后的 policy 字符串**，不是原始 JSON 字节。

解码 token 第三段可核对 policy，例如：

```bash
# 第三段为 encoded_policy，可用 Python 解码查看 scope、deadline 等
```

## 配置了回调时，客户端还能收到响应吗？

能。七牛会**先** POST 到你的 `callbackUrl`，再把你回调服务返回的 **JSON** 转给上传方。  
回调失败时客户端可能收到 **579**，且云上文件可能已存在。

无回调时，客户端直接收到七牛默认 JSON（`hash`、`key`）。
