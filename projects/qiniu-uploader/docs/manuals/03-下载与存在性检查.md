# 03 — 下载与存在性检查

## 前提

1. 已知上传成功返回的 **key**（与上传时一致）。
2. 使用**绑定到该 bucket 的访问域名**（不是上传域名）。
3. 私有空间需 **AK/SK** 在服务端生成带 `e` 与 `token` 的 URL。

### bizdl 已验证组合

| 项 | 值 |
|----|-----|
| bucket | `bizdl-airdroid-com` |
| 上传域名 | `https://up-z2.qiniup.com` |
| 访问域名 | `https://bizdl.airdroid.com` |

`air-test-qn.airdroid.com` / `.cn` 等若未绑定到 `bizdl-airdroid-com`，会 **404**，与 token 是否正确无关。

---

## 私有下载 URL 形态

```
https://<访问域名>/<编码后的 key>?e=<Unix 过期时间>&token=<AK>:<HMAC 签名>
```

- **GET**：下载文件正文。
- **HEAD**：只查是否存在（不拉 body）。
- **浏览器**：在 `e` 过期前，把完整 URL 粘贴到地址栏即可下载。

签名由服务端用 SK 对「`https://域名/路径?e=...`」整串做 HMAC-SHA1，**不是** uploadToken。

## 检查是否存在

```bash
python3 qiniu_check_object_exists.py \
  --base-url "https://bizdl.airdroid.com" \
  --key "videos/task123/device456/滴滴电子发票.pdf" \
  --private \
  --access-key "$QINIU_ACCESS_KEY" \
  --secret-key "$QINIU_SECRET_KEY"

//demo
python3 qiniu_check_object_exists.py   --url "https://up-z2.qiniup.com/videos/task123/device456/滴滴电子发票.pdf"    

python3 qiniu_check_object_exists.py \
  --base-url "https://bizdl.airdroid.com" \
  --key "videos/task123/device456/滴滴电子发票.pdf" \
  --private \
  --access-key "fpWzUFhgkUGVEph0rXeBVu6Y4hfMhuZ5qXqHYjJi" \
  --secret-key "1AsXkvDBlfVrl1NU89Q7InrYZ9GMjGu8Jlnefuqk"   
```

成功示例：

```json
{
  "exists": true,
  "status": 200,
  "url": "https://bizdl.airdroid.com/videos/.../....pdf?e=...&token=..."
}
```

### 等价 curl（HEAD）

先用脚本拿到 `url`，再：

```bash
curl -I 'PASTE_SIGNED_URL_HERE'
```

`200` / `204` 一般表示存在；`404` 表示该域名下没有此 key。

## 下载到本地

```bash
python3 qiniu_download_object.py \
  --base-url "https://bizdl.airdroid.com" \
  --key "videos/task123/device456/滴滴电子发票.pdf" \
  --private \
  --access-key "$QINIU_ACCESS_KEY" \
  --secret-key "$QINIU_SECRET_KEY" \
  --output "/tmp/滴滴电子发票.pdf"

//demo
python3 qiniu_download_object.py \
  --base-url "https://bizdl.airdroid.com" \
  --key "videos/task123/device456/滴滴电子发票.pdf" \

  --base-url "https://bizdl.airdroid.com" \
  --key "videos/task123/device456/滴滴电子发票.pdf" \
```

### 等价 curl（GET）

```bash
curl -L -o "/tmp/滴滴电子发票.pdf" \
  'PASTE_SIGNED_URL_HERE'
```

`-L` 用于跟随 302 等跳转（若域名配置了跳转）。

---

## 不要用上传域名做检查 / 下载

| 操作 | 域名 | 结果 |
|------|------|------|
| HEAD `up-z2.qiniup.com/...` | 上传域 | 可能 **200 误判**「存在」 |
| GET 同上 | 上传域 | **405** Method Not Allowed |
| GET `bizdl.airdroid.com/...?e&token` | 访问域 | 正确方式 |

可靠的存在性检查 = **访问域名 +（私有则）签名 URL + HEAD**。

---

## URL 中的中文

脚本会对 path 做百分号编码；`--url` 传入含中文的完整 URL 时也会自动编码。  
若手写 curl，path 需 UTF-8 百分号编码，例如 `滴滴` → `%E6%BB%B4%E6%BB%B4`。

---

## 公开空间（仅作对比）

若 bucket 为公开读，可不带 `--private`：

```bash
python3 qiniu_check_object_exists.py \
  --base-url "https://你的公开域名" \
  --key "videos/task123/a.pdf"
```

生产环境 bizdl 一般为私有，仍应走 `--private`。
